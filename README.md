# Telegram Account AI Agent

Агент работает через обычный Telegram-аккаунт на Telethon и отвечает через Google Gemini API.

## Быстрый запуск

```powershell
.\.venv\Scripts\Activate.ps1
python main.py
```

Если окружение нужно пересоздать:

```powershell
py -3.11 -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python main.py
```

При первом запуске Telethon попросит номер телефона, код Telegram и, если включено, пароль 2FA.

## Поведение

- В личных сообщениях агент отвечает на все входящие сообщения.
- В личных сообщениях агент прочитывает входящие сообщения перед ответом.
- В группах агент отвечает на `.ai <текст>`, упоминание аккаунта или обращение по имени `вилсон` / `вильсон`.
- Фото с подписью обрабатываются вместе с текстом. Если модель не умеет смотреть картинки, агент скажет об этом коротко.
- Если админ отправляет фото с просьбой поставить/поменять аватарку, агент ставит именно это фото.
- Если в команде есть `удалить старую`, агент ставит новую аватарку и удаляет предыдущую. Без этой фразы новая аватарка просто добавляется.
- Может менять имя/никнейм, фамилию, юзернейм и описание профиля.
- Может отправлять сообщения по просьбе админа и вступать в чаты/каналы по `@username` или ссылке.
- Может по просьбе админа читать историю чата и проверять, был ли ответ после исходящего сообщения; содержимое переписки отправляется только админу.
- В автономном режиме может писать только в уже существующие диалоги, а также удалять диалог, блокировать или разблокировать пользователя при очевидной причине.
- Пока агент работает, он периодически обновляет online-статус аккаунта.
- Админ: `634552356`.
- Короткая локальная память хранится отдельно для каждого чата в `data/memory`.
- Долгая память хранится в PostgreSQL: чаты, люди, сообщения, заметки и поведенческие профили.
- Системный промпт лежит в `prompts/system.txt` и сейчас специально пустой.

## Управление аккаунтом

Вилсон может менять пространство своего Telegram-аккаунта по явной просьбе админа:

- имя профиля;
- фамилию;
- био;
- юзернейм;
- аватарку из сообщения, на которое ты ответил;
- удалять текущую аватарку;
- показывать текущий профиль.

Примеры:

```text
Вилсон, поменяй имя профиля на Wilson
Вильсон, поставь био: личный ии-агент
Вилсон, поменяй юзернейм на my_wilson_ai
```

Чтобы поставить аватарку, ответь на сообщение с картинкой:

```text
Вилсон, поставь это на аватарку
```

Или отправь фото сразу с подписью:

```text
Вилсон, поменяй аватарку на это фото
```

```text
Вилсон, поменяй аватарку на это фото и удали старую
```

Другие действия:

```text
Вилсон, поменяй описание на обычный чел из чата
Вилсон, поменяй никнейм на Wilson
Вилсон, поменяй юзернейм на my_wilson_ai
Вилсон, напиши @username привет
Вилсон, ответил ли @username
Вилсон, прочитай чат @username
Вилсон, удали чат @username
Вилсон, кинь @username в чс
Вилсон, достань @username из чс
Вилсон, зайди в https://t.me/channelname
```

Действия профиля заблокированы для всех, кроме админа.

## Настройки

Основные настройки находятся в `.env`:

```env
RESPOND_TO_ALL_PRIVATE=true
TRIGGER_PREFIX=.ai
TRIGGER_NAMES=вилсон,вильсон,вилка,wilson
DATABASE_URL=postgresql://...
MAX_HISTORY_MESSAGES=30
MAX_TOKENS=1600
TEMPERATURE=0.7
MESSAGE_BATCH_DELAY=4.0
MAX_BATCH_MESSAGES=25
MAX_PASSIVE_CONTEXT_MESSAGES=80
DB_CONTEXT_MESSAGES=35
AUTONOMOUS_ACTIONS_ENABLED=true
MAX_AUTONOMOUS_MESSAGES=4
SEND_MESSAGE_DELAY=2.0
PEER_FLOOD_COOLDOWN=900
KEEP_ONLINE=true
ONLINE_PING_INTERVAL=45
GOOGLE_API_KEY=...
GOOGLE_MODEL=gemini-2.5-flash
GOOGLE_FALLBACK_MODELS=gemini-flash-latest,gemini-3-flash-preview,gemini-2.5-flash-lite,gemini-flash-lite-latest,gemini-3.1-flash-lite-preview,gemini-2.0-flash,gemini-2.0-flash-001,gemini-2.0-flash-lite,gemini-2.0-flash-lite-001,gemma-4-31b-it,gemma-4-26b-a4b-it,gemma-3-27b-it,gemma-3-12b-it,gemma-3n-e4b-it,gemma-3n-e2b-it,gemma-3-4b-it,gemma-3-1b-it,gemini-2.5-pro,gemini-pro-latest,gemini-3-pro-preview,gemini-3.1-pro-preview
GOOGLE_PROXY=
GOOGLE_TIMEOUT=90
GOOGLE_RETRIES=3
```

Если в консоли появляется `All connection attempts failed`, значит Python не может
достучаться до Google Gemini. Проверь VPN/интернет или укажи прокси:

```env
GOOGLE_PROXY=http://127.0.0.1:7890
```

Для SOCKS-прокси нужно поставить поддержку socks:

```powershell
pip install "httpx[socks]"
```

Если появляется `429` или `503`, бесплатная модель временно уперлась в лимит или перегруз.
Агент сначала попробует `GOOGLE_MODEL`, потом модели из `GOOGLE_FALLBACK_MODELS`.

`DATABASE_URL` включает PostgreSQL-память. Агент пишет туда сообщения, людей и чаты, а перед ответом подмешивает `DB_CONTEXT_MESSAGES` последних релевантных записей и заметки памяти. `MESSAGE_BATCH_DELAY` задает паузу ожидания после сообщения: если за это время прилетят еще сообщения в тот же чат, агент обработает их одним пакетом. В группах сообщения без явного вызова сохраняются как пассивный контекст, но агент молчит до обращения по имени/юзернейму. `MAX_PASSIVE_CONTEXT_MESSAGES` ограничивает объем локального пассивного контекста. `AUTONOMOUS_ACTIONS_ENABLED` разрешает модели по своему решению отправлять сообщения через инструменты, но только в админском контексте и с лимитом `MAX_AUTONOMOUS_MESSAGES` за один ответ. `SEND_MESSAGE_DELAY` ставит паузу между несколькими исходящими сообщениями, а `PEER_FLOOD_COOLDOWN` временно блокирует отправки после Telegram flood-защиты.
